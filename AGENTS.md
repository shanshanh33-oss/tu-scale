# AGENTS.md

本文件适用于当前仓库及其全部子目录。

本项目已经包含可以正常使用的图片放大、图片压缩、商品图规范化、AI 抠图、统计和反馈功能。修改代码时必须优先保护已有功能、用户数据和生产环境安全。

## 基本规则

1. 修改前必须先阅读 `PROJECT.md`、`TODO.md` 和相关源文件。
2. 修改必须以现有代码为依据，不得凭空假设已有接口、服务或数据结构。
3. 未经用户明确确认，不得进行与当前任务无关的重构。
4. 不得删除、覆盖或回退用户已有修改。
5. 工作区存在无关变更时，必须保留并避开这些变更。
6. 优先进行范围明确、可以验证的小修改。
7. 不得使用破坏性 Git 命令处理未确认的文件。
8. 不得提交 `.env.local`、API Key、Token、用户反馈文件或其他敏感信息。

## 不得擅自更换技术栈

未经用户明确确认，禁止：

- 将 React 更换为 Vue、Next.js、Nuxt、Svelte 或其他框架。
- 将 Vite 更换为其他构建系统。
- 将 JavaScript 全量迁移到 TypeScript。
- 引入新的路由框架替换现有 History API 路由。
- 更换 Tailwind CSS 或整体替换样式方案。
- 更换 Cloudflare Pages Functions 和 KV。
- 更换 remove.bg、ONNX Runtime、waifu2x 或现有图片处理方案。
- 引入大型状态管理、组件库或后端框架。

如果当前任务确实需要新依赖或技术调整，必须先说明：

1. 当前方案为什么不能满足需求。
2. 新方案影响哪些文件和部署环境。
3. 新增依赖的体积、维护和安全风险。
4. 是否存在不增加依赖的替代方案。

得到确认后才能实施。

## 修改前必须说明影响范围

开始编辑前，必须向用户简要说明：

1. 准备修改的功能。
2. 涉及的文件。
3. 是否影响前端、Functions、KV、第三方 API 或部署。
4. 可能存在的兼容性和数据风险。
5. 计划执行的验证方式。

如果检查后发现影响范围扩大，必须在继续扩大修改前再次说明。

纯文档、注释或明确的机械修改也应说明涉及文件，但可以简化说明。

## 修改后必须验证

每次修改后必须运行与改动相匹配的检查。

前端或共享代码至少运行：

```bash
cd saas/ecommerce-img
npm run lint
npm run build
```

仅修改 Node CommonJS 服务时，至少运行：

```bash
node --check local-server.cjs
node --check server/remove-bg-server.cjs
node --check server/ai-server.cjs
```

修改 Cloudflare Functions 时，至少对相关 JavaScript 文件运行语法检查，并在条件允许时执行接口测试。

修改图片处理逻辑时，除 lint 和 build 外，还必须说明已经检查的场景，例如：

- JPG、PNG 和透明 PNG
- 单图和批量
- 原尺寸和自定义尺寸
- 裁切开启和关闭
- AI 开启和关闭
- 成功、失败和取消流程

如果因为依赖未安装、网络、密钥或外部服务不可用而无法运行验证，必须：

1. 明确说明未执行的命令。
2. 说明阻塞原因。
3. 不得声称修改已经完全验证。
4. 给出用户可以执行的验证命令。

不得为了让测试通过而删除测试、降低检查标准或静默忽略错误。

## UI 修改规则

UI 修改必须保持现有视觉风格：

- 继续使用当前 Tailwind CSS 风格。
- 保持白色卡片、浅灰背景、圆角、细边框和靛蓝/紫色主色。
- 优先复用现有间距、字号、按钮和状态样式。
- 保持桌面端和移动端响应式布局。
- 不得未经确认整体更换 Logo、品牌色、字体或页面结构。
- 不得引入与现有风格明显冲突的组件库。
- 新交互必须有加载、禁用、成功和失败状态。
- 图片编辑交互需要兼顾鼠标和触摸操作。
- 新按钮和输入框应具备明确文字、可访问名称和键盘操作能力。

修改 UI 后应检查至少以下页面：

- `/`
- `/format-converter`
- `/product-image`
- `/contact`

- 服务端密钥保护

- 超时处理

KV 未配置时，不应默认放开付费 API 调用。
